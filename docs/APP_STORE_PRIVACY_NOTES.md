# App Store Privacy Notes - MVP

Last updated: 2026-05-03

This document maps the current Local Footy Athlete MVP data flows to App Store privacy planning. It is practical release guidance, not legal advice.

## Current Product Position

- The live MVP does not require a user account.
- Auth/account screens and Supabase auth code exist in the repo, but they are not mounted by the live app navigator.
- Profile exposes Privacy Policy, Terms of Use, Support, and Full reset.
- Delete Account is not shown because the live MVP has no account creation or login flow.
- Full reset clears local app data and returns the app to onboarding.

## Third-Party Processors

- Supabase: edge functions for coach intent and coach chat/program generation. Some dormant service code also targets Supabase database/auth, but live MVP routing does not expose account auth.
- Anthropic / AI provider: called from Supabase edge functions for coach intent classification, coach replies, and program generation support.
- YouTube / Google: exercise videos are embedded from `youtube-nocookie.com` only when a user opens a video.
- Email provider: support and feedback use `mailto:` links, so the user's email app sends the message.
- Expo / React Native platform services: app runtime/build stack. No explicit Expo analytics or crash reporting SDK was found in live code.

## iOS Privacy Manifest

`ios/LocalFootyAthlete/PrivacyInfo.xcprivacy` declares:

- Tracking: false
- Required reason APIs:
  - UserDefaults, for app state persistence through React Native/Expo storage
  - File timestamp, disk space, and system boot time, inherited from native/runtime dependencies
- Collected data types:
  - Name
  - Health
  - Fitness
  - Other User Content
- Purposes:
  - App Functionality
  - Product Personalization
- Linked to user: true
- Used for tracking: false

Important distinction: the privacy manifest is not the full App Store Connect privacy questionnaire. The questionnaire still needs manual answers for all data categories, including support email handling and backend/AI processors.

## Suggested App Store Privacy Questionnaire Answers

Use these as a starting point for App Store Connect. Final answers should be confirmed by the founder before submission.

### Data Used To Track The User

- No.
- No advertising tracking, cross-app tracking, or data broker sharing was found.

### Contact Info

- Name: Yes, collected.
  - Purpose: App Functionality, Product Personalization
  - Linked to user: Yes
  - Tracking: No
- Email Address: Not collected by the app account flow because there is no live account flow. However, if users email support from the app, the founder receives their email address.
  - Founder decision: Decide whether to disclose support email address collection under Contact Info for App Store labels.

### Health and Fitness

- Health: Yes.
  - Includes injury context, soreness, fatigue, pain severity, body measurements, and training readiness context.
  - Purpose: App Functionality, Product Personalization
  - Linked to user: Yes
  - Tracking: No
- Fitness: Yes.
  - Includes program setup, schedule, game day, workouts, training completion, performed weights, and training preferences.
  - Purpose: App Functionality, Product Personalization
  - Linked to user: Yes
  - Tracking: No

### User Content

- Other User Content: Yes.
  - Includes coach chat messages and free-text training context.
  - Purpose: App Functionality, Product Personalization
  - Linked to user: Yes
  - Tracking: No
- Customer Support: Yes if App Store labels treat the in-app mailto support flow as collection.
  - Purpose: Customer Support / App Functionality
  - Linked to user: Yes if sent
  - Tracking: No

### Identifiers

- User ID: No live account ID is collected in the current MVP.
- Device ID: No explicit collection found.

### Purchases

- No live purchase/subscription flow found.
- RevenueCat is installed as a dependency but no live import/use was found. If subscriptions are enabled later, update this document and App Store labels.

### Usage Data

- Product Interaction: No analytics SDK found.
- Advertising Data: No.
- Other Usage Data: No explicit off-device collection found.

### Diagnostics

- Crash Data: No crash reporting SDK found.
- Performance Data: No analytics/performance reporting SDK found.
- Other Diagnostic Data: No explicit off-device collection found.

### Sensitive Info

- Do not select Sensitive Info unless legal review decides injury/pain context should be classified there. Current recommendation is to disclose it under Health and Fitness.

## External URLs Needed

App Store Connect requires stable external URLs:

- Privacy Policy URL
- Support URL

No stable external privacy/support URL is configured in the live app. Do not submit fake URLs.

Minimal acceptable options:

- A simple Carrd page with Privacy Policy and Support sections
- A Notion public page with stable URLs
- GitHub Pages with `/privacy` and `/support`
- A minimal `localfootyathlete.com/privacy` and `localfootyathlete.com/support`

Support URL can be a simple page that lists the support email and expected response path. The support email currently used in app code is `one22gym@gmail.com`.
