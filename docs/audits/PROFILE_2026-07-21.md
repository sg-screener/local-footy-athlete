state" (Developer Tools) and "Full reset" (Danger Zone) were seen and read
but not tapped. Both are one-way resets — "Full reset" explicitly "Returns
to onboarding" — and Onboarding/Auth are separate, not-yet-audited areas in
the `QA_RUNBOOK.md` order. Tapping either here would risk landing this
session in a state this audit isn't equipped to recover from or verify,
for no incremental coverage beyond what their own labels already state
plainly. Recommend exercising them specifically during the Onboarding
and/or Auth area audits, where a fresh-install-equivalent state is already
the expected starting point.

the coach" stays in a native form flow, unlike Home V2's "Busy or away this
week?", which routes into Coach chat for what looks like the same class of
setup change. Worth flagging to whoever owns the Coach Architecture
Escalation Rule reassessment as a concrete example of two different
ownership models for what a user would expect to be the same kind of edit.
